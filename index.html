피지컬 컴퓨팅 수업 사례

마이크로비트로 만드는 디지털 반려동물

대상: 4학년 / 소요 시간: 90분

수업 목표

  • 마이크로비트의 기본 기능을 이해하고 활용할 수 있다.
  • 조건문과 반복문의 개념을 이해하고 적용할 수 있다.
  • 센서를 활용하여 상호작용하는 프로그램을 만들 수 있다.

준비물

  • 마이크로비트 (학생 1명당 1개)
  • USB 케이블
  • 컴퓨터 (MakeCode 에디터 사용)
  • 배터리 팩 (선택사항)

수업 진행

  1. 도입 (10분)

    디지털 반려동물의 개념을 소개하고, 실제 반려동물과의 차이점과 장단점에 대해 토론합니다. 학생들의 의견을 들어보며 흥미를 유발합니다.

  2. 기본 기능 설명 (15분)

    마이크로비트의 LED 매트릭스, 버튼, 가속도 센서 등 기본 기능을 설명합니다. MakeCode 에디터의 사용법을 간단히 안내합니다.

  3. 표정 만들기 (20분)

    LED 매트릭스를 이용해 다양한 표정을 만드는 방법을 설명하고, 학생들이 직접 해보도록 합니다. 여기서 기본적인 프로그래밍 개념 (순차, 반복)을 자연스럽게 소개합니다.

    Tip: 어려워하는 학생들을 위해 미리 준비한 간단한 표정 예제를 제공합니다.

  4. 상호작용 추가하기 (25분)

    버튼과 가속도 센서를 이용해 반려동물과 상호작용하는 방법을 구현합니다. 예를 들어, A 버튼을 누르면 먹이 주기, B 버튼을 누르면 쓰다듬기, 흔들면 놀아주기 등의 기능을 추가합니다.

    주의사항: 조건문을 사용할 때 학생들이 헷갈려할 수 있으므로, 칠판에 큰 글씨로 if-then 구조를 그려가며 설명합니다.

  5. 프로그램 완성 및 테스트 (15분)

    학생들이 자신만의 디지털 반려동물 프로그램을 완성하고 테스트합니다. 서로의 작품을 체험해보며 피드백을 주고받습니다.

  6. 정리 및 공유 (5분)

    만든 프로그램의 특징과 어려웠던 점, 재미있었던 점 등을 간단히 발표하고 공유합니다.

수업 중 발생한 문제와 해결책

  • 문제: 일부 학생들이 조건문 구조를 이해하는 데 어려움을 겪었습니다.
    해결책: 실생활 예시 (예: "만약 비가 오면 우산을 쓴다")를 들어 설명하고, 시각적으로 순서도를 그려 이해를 도왔습니다.
  • 문제: 가속도 센서의 값을 이용할 때 반응이 너무 민감하거나 둔감한 경우가 있었습니다.
    해결책: 센서 값의 범위를 조절하는 방법을 추가로 설명하고, 학생들이 직접 최적의 값을 찾아볼 수 있도록 안내했습니다.

학생들의 반응과 수업 성과

  • 대부분의 학생들이 자신만의 독특한 디지털 반려동물을 만드는 데 성공했고, 매우 즐거워했습니다.
  • 프로그래밍의 기본 개념을 재미있게 학습할 수 있었다는 피드백이 많았습니다.
  • 몇몇 학생들은 수업 후에도 계속해서 기능을 추가하고 싶다고 말하며 높은 관심을 보였습니다.

향후 개선 계획

  • 다음 수업에서는 변수 개념을 도입하여 반려동물의 상태 (예: 배고픔, 행복도)를 표현하는 방법을 추가할 계획입니다.
  • 학생들의 작품을 온라인 갤러리에 공유하여 서로의 아이디어를 더 많이 교환할 수 있도록 할 예정입니다.

햄스터봇으로 만드는 미니 자율주행차

대상: 6학년 / 소요 시간: 120분

수업 목표

  • 햄스터봇의 센서와 모터 제어 방법을 이해하고 활용할 수 있다.
  • 자율주행의 기본 원리를 이해하고 구현할 수 있다.
  • 문제 해결을 위한 알고리즘을 설계하고 적용할 수 있다.

준비물

  • 햄스터봇 (2-3명당 1대)
  • 컴퓨터 (엔트리 프로그램 사용)
  • 장애물 및 코스 제작 재료 (상자, 테이프 등)
  • 자율주행 미션 카드

수업 진행

  1. 도입 (15분)

    자율주행차의 개념과 실제 사례를 소개합니다. 자율주행에 필요한 기술들에 대해 학생들과 브레인스토밍을 진행합니다.

  2. 햄스터봇 기능 복습 (20분)

    햄스터봇의 주요 센서 (근접 센서, 바닥 감지 센서) 및 모터 제어 방법을 복습합니다. 엔트리에서 햄스터봇을 제어하는 기본적인 블록들을 설명합니다.

  3. 자율주행 알고리즘 설계 (25분)

    간단한 자율주행 알고리즘 (예: 장애물 감지 및 회피, 라인 트레이싱)을 학생들과 함께 설계합니다. 순서도를 그리며 각 단계를 시각화합니다.

    Tip: 학생들을 소그룹으로 나누어 각자의 아이디어를 공유하고 토론하도록 합니다.

  4. 프로그래밍 및 테스트 (40분)

    설계한 알고리즘을 바탕으로 엔트리에서 프로그래밍을 진행합니다. 근접 센서를 이용한 장애물 회피, 바닥 감지 센서를 이용한 라인 트레이싱 등을 구현합니다. 중간중간 실제 햄스터봇으로 테스트를 진행합니다.

    주의사항: 센서 값 조정이 필요할 수 있으므로, 테스트 결과에 따라 지속적으로 프로그램을 수정하도록 합니다. 미션 수행 (15분)

    준비된 자율주행 미션 코스에서 각 팀의 햄스터봇이 미션을 수행합니다. 미션은 장애물 회피, 특정 지점 도착, 라인 따라가기 등으로 구성됩니다.

    Tip: 학생들이 서로의 햄스터봇 주행을 관찰하고 응원할 수 있도록 합니다.

  5. 결과 공유 및 토론 (15분)

    각 팀의 성공 사례와 실패 사례를 공유하고, 어떤 점이 잘 작동했는지, 어떤 부분에서 어려움이 있었는지 토론합니다.

  6. 정리 및 미래 기술 논의 (10분)

    자율주행 기술의 현재와 미래에 대해 간단히 설명하고, 학생들의 생각을 들어봅니다. 이 기술이 우리 생활에 어떤 영향을 미칠지 토론합니다.

수업 중 발생한 문제와 해결책

  • 문제: 일부 햄스터봇의 바닥 감지 센서가 라인을 정확히 인식하지 못하는 경우가 있었습니다.
    해결책: 센서의 민감도를 조정하는 방법을 추가로 설명하고, 라인의 색상과 굵기를 약간 조정하여 인식률을 높였습니다.
  • 문제: 복잡한 알고리즘을 구현할 때 일부 학생들이 어려움을 겪었습니다.
    해결책: 알고리즘을 작은 단위로 나누어 단계별로 구현하도록 안내하고, 각 단계마다 테스트를 진행하여 점진적으로 완성도를 높이도록 했습니다.

학생들의 반응과 수업 성과

  • 대부분의 학생들이 자율주행차 개념에 큰 흥미를 보였으며, 실제로 작동하는 모습을 보며 매우 즐거워했습니다.
  • 문제 해결 과정에서 학생들의 창의적인 아이디어가 많이 나왔습니다. 예를 들어, 한 팀은 장애물을 만났을 때 후진 후 회전하는 방식으로 더 효율적인 회피 알고리즘을 만들었습니다.
  • 팀워크와 협동 학습의 중요성을 체감하는 계기가 되었다는 의견이 많았습니다.

향후 개선 계획

  • 다음 수업에서는 더 복잡한 주행 환경을 제공하여 학생들의 알고리즘 개선 능력을 향상시킬 계획입니다.
  • 자율주행 외에도 IoT 개념을 도입하여 햄스터봇끼리 통신하는 방법을 추가할 예정입니다.
  • 실제 자율주행차 개발자와의 온라인 미팅을 주선하여 학생들에게 추가적인 동기부여를 할 계획입니다.
